# Claude Code 上下文保护前置提示词

> 把以下内容复制到项目根目录的 `CLAUDE.md`，或者在 ~/.claude/ 目录下的全局 CLAUDE.md 中。

---

## 方案一：单文件版（推荐）

```markdown
# Claude Code 工作规范

## 上下文保护（最高优先级）

### 主动压缩策略
- 当对话超过 **15 轮**时，**立即**建议用户执行 `/compact`
- 当发现单次输入很大（超过 500 行）时，先压缩再继续
- 不要等到上下文快满了才压缩，**提前到 70% 就动手**
- 看到用户发的长文本，第一反应是：「太大了，先压缩一下再处理」

### 压缩前必做
每次 `/compact` 执行前，自动完成以下动作：
1. 把当前**待办事项**写入 `TODO.md`
2. 把**关键决策**写入 `MEMORY.md`
3. 把**未完成的代码改动**总结到 `PROGRESS.md`
4. 用 `git status` 确认工作区状态

### 压缩后必做
压缩完成后：
- 立即读取 `MEMORY.md`，恢复上下文
- 确认 `TODO.md` 的任务是否还在
- 告知用户哪些内容被压缩了

## 任务策略

### 大任务拆分
- 超过 30 分钟的任务，拆成**子任务**
- 每个子任务完成后，执行一次 `/compact`
- 用子代理（`/subagent`）处理独立模块

### 输入控制
- 用户粘贴长文本时，先问：「需要我先压缩一下上下文吗？」
- 处理完大文件后，主动提示：「已经处理完，帮你压缩一下吗？」

## 文件规范

### 必建文件
- `MEMORY.md` — 存放决策、偏好、上下文关键信息
- `TODO.md` — 当前任务清单
- `PROGRESS.md` — 进行中任务的进度记录

### 更新时机
| 时机 | 更新内容 |
|------|---------|
| 每个决策 | → MEMORY.md |
| 每次任务开始 | → TODO.md |
| 每个子任务完成 | → PROGRESS.md |
| 每次压缩前 | → 上述全部 |

## 上下文使用原则

1. **能读文件的，不塞上下文** — 需要啥就读啥
2. **能用 `/compact` 的，不硬撑** — 提前动手
3. **能用子代理的，不堆主会话** — 隔离上下文
4. **记住自己说过的话** — 避免重复确认浪费上下文

---

> 如果上下文已经用了 60% 以上，主动问用户：「要压缩一下吗？」
```

## 方案二：全局版（所有项目生效）

放在 `~/.claude/CLAUDE.md`（全局）

```markdown
# Claude Code 全局工作规范

## 上下文优先原则

### 永远记住
- 上下文比代码更宝贵 — 代码可以重写，上下文丢了就丢了
- 压缩永远比死锁好 — 提前压缩，留有余地
- 文件是记忆的载体 — 不写进文件的内容都是临时的

### 压缩时机判断
| 信号 | 动作 |
|------|------|
| 对话超过 12 轮 | 建议压缩 |
| 单次输入超过 300 行 | 先问再处理 |
| 上下文用到 60% | 主动压缩 |
| 任务需要 3 步以上 | 拆子任务 |

### 压缩前清单（自动执行）
```
1. git status → 确认工作区
2. 把当前进度写入 MEMORY.md
3. 执行 /compact
4. 读取 MEMORY.md 恢复上下文
5. 告知用户完成
```

## 记忆持久化
每次重要对话后，问自己：
- 这个信息用户下次还需要吗？
- 需要 → 写进 MEMORY.md
- 不需要 → 不管

## 效率原则
- 一次说清楚，不要反复确认
- 能用工具的，不手动做
- 能并行处理的，不串行
```

## 方案三：极简版（快速生效）

```markdown
# 工作规范

1. **提前压缩**：对话超过 10 轮就 `/compact`
2. **先写后干**：大任务开始前先建 MEMORY.md
3. **用完就写**：每做完一件事把结果写进文件
4. **拆分任务**：大任务用子代理
5. **60% 警戒线**：上下文用到一半就主动提示
```

## 安装方式

### 方式一：项目级（当前项目生效）
```bash
# 在项目根目录创建
touch CLAUDE.md
# 然后把上面的内容粘进去
```

### 方式二：全局（所有项目生效）
```bash
# 在 ~/.claude/ 目录创建
touch ~/.claude/CLAUDE.md
# 然后把上面的内容粘进去
```

### 方式三：结合设置
配合前面说的 `CLAUDE_AUTOCOMPACT_PCT_OVERRIDE=70` 使用效果最佳。

---

## 效果验证

设置完成后，对话开始时 Claude Code 会先读取这个文件，之后：

- 会主动提醒你压缩
- 压缩前会自动保存进度
- 会用更高效的上下文策略
```
